## 4. 已验证的 CFG 技巧 (2026-07-05 实测)

### 静态 3D 点云 — 纯 CFG 实现
**关键发现**: 不改固件，仅调 2 行 CFG，可从 OOB demo 获得静态物体的 3D 坐标。

```cfg
clutterRemoval 0                    # 保留零多普勒 DC 信号
cfarProcCfg 0 2 8 4 3 0 3.0 0      # range CFAR 阈值 9.0→3.0 dB
cfarProcCfg 1 2 4 2 2 1 3.0 0      # doppler CFAR 阈值 9.0→3.0 dB
```

| 指标 | 值 |
|------|-----|
| 检测点/帧 | ~45 (含静态+动态) |
| 静态点/帧 | ~12 (doppler=0) |
| 帧率 | 10 fps |
| 60秒累积静态点 | ~6700 |
| 输出格式 | (x, y, z, doppler) float32 |
| 验证结果 | 桌/椅/墙 位置和高度正确 |

### 已知限制
| 限制 | 原因 |
|------|------|
| 角分辨率 29° | 硬件限制，16 虚拟天线 |
| CFAR 不可关闭 | HWA 硬连线，只能调参数 |
| threshold=0.0 导致崩溃 | 固件不接受 |
| rangeAzimuthHeatMap 不支持 | L-SDK 限制 |
| guard/window 极值导致 DPU error:-2832 | 参数组合约束 |

### OOB 默认 Profile 对比

| Profile | TX×RX | Chirps | CFAR Threshold | MIMO |
|---------|-------|--------|----------------|------|
| profile_4T4R_tdm.cfg | 4×4=16 | 64 | **9.0 dB** | TDM |
| profile_3T4R_tdm.cfg | 4×3=12 | 48 | **6.0 dB** | TDM |
| profile_2T4R_bpm.cfg | 4×2=8 | 64 | **9.0 dB** | BPM |
| **profile_extreme_cfar.cfg** | 4×4=16 | 64 | **3.0 dB** | TDM |
| **profile_static_map.cfg** | 4×4=16 | 64 | **2.0 dB** | TDM |

### 静态地图优化配置 (profile_static_map.cfg)
基于 Area Scanner 参考 + 实测经验的优化版本:

```cfg
cfarProcCfg 0 2 8 4 3 0 2.0 0      # range CFAR 阈值 9.0→2.0 dB (更激进)
cfarProcCfg 1 2 4 2 2 1 2.0 0      # doppler CFAR 阈值 9.0→2.0 dB
cfarFovCfg 0 0.15 6.5              # range FOV 收紧到房间尺寸
aoaFovCfg -70 70 -70 70            # 角度 FOV 扩大到 ±70°
clutterRemoval 0                    # 保留静态
```
优化要点:
- **CFAR 2.0 dB**: 比 3.0 更灵敏，预期更多静态点。若不稳定回退到 2.5
- **Range FOV 0.15-6.5m**: 过滤远处噪声 (原 0.25-9.0m)
- **AoA FOV ±70°**: 捕获更多侧面物体 (原 ±60°)
- 若 2.0 dB 崩溃，安全回退: 先试 2.5，再试 3.0

## 5. 产品架构: Server-MUSIC + 三模式运行 (2026-07-06)

### 5.1 核心原则

**芯片是带 WiFi 的传感器，不是计算平台。所有智能放 Server，随时迭代。**

- 芯片不做 CFAR / AoA FFT 以外的计算（标准 OOB 管线不变）
- 唯一固件改动：新增 TLV type 输出 zero-Doppler antenna complex data
- MUSIC 超分辨、语义建图、Fall 判断、stability 系数全在 Server
- 算法升级 = 改 Python 重启，不碰硬件

### 5.2 三模式运行

| 模式 | 时机 | UART 数据 | Server 处理 |
|------|------|-----------|-------------|
| **夜间标定扫描** | 凌晨无人 (如 3am) | antenna complex, 分层, K=50+ | MUSIC 全房间高清图 → 语义建图 |
| **日间监控** | 全天 | 标准点云 8KB/s | zone 能量比较 → 异常检测 |
| **事件 Gaze** | ghost/fall/lost/stillbox | antenna complex, 指定区域, 10s | MUSIC K=100 → 精细判断 |

#### 模式 1: 夜间标定扫描

凌晨无人，大把时间。缓慢分 bin 扫全房间，每个 range bin 累积 K=50 snapshot。

```
利用静态不变性: 
  静态目标的 antenna response 帧间不变 (只有噪声独立变化)
  → 不同帧的同一 bin 可作为独立 snapshot 累积
  → 时间换空间，不需要一帧发所有 bin

数据流:
  HWA: Range FFT → Doppler FFT (每帧自动完成)
  固件: 取 Doppler bin 0 (零多普勒) → 取当前轮到的 range bin 段
        → 打包 TLV → UART
  ESP32: WiFi → Server

分层传输 (7GHz 满带宽, 405 有效 bins):
  每帧: ~100 bins × 16 ant × int16 complex = 6.8 KB
  分 4 层, K=50: 200 帧 = 20 秒完成全房间扫描
  K=500 (极致): 2000 帧 = 3.3 分钟

每天重复价值:
  Day 1:  baseline → 初始语义图
  Day 2:  diff → 检测家具变化
  Day 7:  stability 系数收敛
  Day 30: 完整空间语义模型
```

**Doppler FFT 的零多普勒 bin = 64 chirp 相干积分 → SNR +18dB。** Server 拿到的每个 snapshot 质量远高于单 chirp 原始数据。

#### 模式 2: 日间监控

```
标准 OOB 管线: CFAR → FFT AoA → 点云 (x,y,z,doppler)
数据量: ~50 pts × 16B × 10fps = 8 KB/s
Server: 点云 → zone 能量累积 → 与夜间基线比较
        zone 比较用 stability 系数加权 (椅子区高阈值, 墙低阈值)
不跑 MUSIC, 不传 antenna data, DSP 负载 3ms/帧
```

#### 模式 3: 事件 Gaze

```
触发条件: zone 能量异常 / doppler 检测下坠 / 轨迹丢失 / stillbox 超时
动作:
  ① ESP32 发 sensorStop → 缩窄 CFG (cfarFovCfg + aoaFovCfg) → sensorStart
  ② 指定区域 ~20 range bins → 不用分层, 一帧全发
  ③ 10s × 10fps = 100 snapshots → K=100
  ④ 上传 128KB → Server MUSIC 精细判断

凝视 CFG 示例 (ObjectC @ Range=4.5m, Az=30°, El=20°):
  cfarFovCfg 0 3.5 5.5          # range ±1m
  aoaFovCfg 15 45 5 35          # 30°×30°

10 秒代价极低:
  fall 场景: 人躺在地上几分钟, 10s 判断不影响任何结果
  ghost 场景: 确认是否真人, 10s 远快于 30s staleness TTL
  lost 场景: 搜索丢失轨迹, 10s 在 10min blind 窗内微不足道
```

### 5.3 Chirp 参数优化 — 最大化 MUSIC 信息量

**原则: 芯片不算，Server 算。最大化信息密度，不考虑 DSP 约束。**

```
优化方向:
  1. chirp 带宽最大化 → 7GHz → range 分辨率 2.1cm (当前 9.2cm, 4.4×)
  2. ADC 采样数最大化 → 1024 → 更多 range bins
  3. chirp slope 由带宽和 ADC 采样时间决定
```

| 方案 | 带宽 | slope | ADC | ΔR | bins | 全扫时间 (K=50) |
|------|------|-------|-----|-----|------|-----------------|
| 当前 | 1.6 GHz | 13.1 MHz/μs | 256 | 9.2 cm | 94 | 5s |
| 推荐 | 7.0 GHz | 136.7 MHz/μs | 1024 | 2.1 cm | 405 | 20s |

UART 带宽约束: 安全上限 73.7 KB/s (1.25Mbps, ~60% 效率)
- 分层传输: 每帧 ~100 bins × 16 ant × 4B = 6.8 KB → UART 用 92% @10fps
- 点云 + antenna data 不同时发 (模式 2 和模式 1/3 互斥)

### 5.4 MUSIC 分辨率实测 (仿真验证)

**4T4R 16 虚拟天线, int16 complex, SNR 20dB:**

| K (snapshots) | 帧数 (奇偶分层) | 时间 @10fps | 角度分辨率 | @3m 横向精度 | vs FFT 29° |
|---------------|----------------|-------------|-----------|-------------|------------|
| 5 | 10 | 1.0s | ~10° | 0.52m | 2.9× |
| 10 | 20 | 2.0s | ~9° | 0.47m | 3.2× |
| 20 | 40 | 4.0s | ~8° | 0.42m | 3.6× |
| 50 | 200 | 20s | ~6° | 0.31m | 4.8× |
| 100 | 400 | 40s | ~4° | 0.21m | 7.3× |

K=5→50 提升有限 (10°→6°)，瓶颈是 4×4=16 天线的物理孔径。
K=50 (夜间标定) 已拿到大部分收益。
K=100 (事件 Gaze) 进一步提升到 4°，用于区分人/椅子/ghost。

### 5.5 固件改动 — 唯一改动点

```c
// 在 Doppler FFT 输出后 (HWA 已完成), AoA FFT 之前:
// 取 zero-Doppler bin (dopplerIdx=0) 的 antenna data
// int16 量化 (ADC 本身 12-bit, int16 无损)
// 打包为新 TLV type

// TLV 格式:
//   type:    TLV_RANGE_ANTENNA (8)
//   header:  start_bin (uint16) + num_bins (uint16) = 4 bytes
//   payload: num_bins × 16 antennas × int16_complex (4B/antenna) 
//   示例:    100 bins → 4 + 100×64 = 6,404 bytes

// ESP32 通过 CLI UART 控制:
//   rangeAntennaOutput <start_bin> <num_bins> <enable>
//   模式 1/3: enable=1, 指定 bin 范围
//   模式 2:   enable=0, 只发点云
```

**ADC 本身是 12-bit，int16 无损。float32 浪费 2× 带宽无意义。**

### 5.6 Server 端管线 (已实现)

```
已写就的 Python 模块:
  spatial3d/music.py          — MUSIC 算法 (2D spectrum, DOA, refine_angles)
  spatial3d/tlv.py            — TLV 解析 (含 TLV_ANTENNA_COMPLEX type 8)
  spatial3d/complex_collect.py — antenna complex 数据采集
  spatial3d/music_collect.py  — 端到端管线 (采集→MUSIC→保存)

Server 处理流程:
  夜间: 累积 K=50 snapshot/bin → 协方差矩阵 → MUSIC DOA → 高清 3D 体素图
        → 语义分析 (识别墙/床/桌/椅) → stability 系数 → 下发 zone config
  日间: 点云 → zone 能量比较 → 异常检测
  事件: gaze 数据 → MUSIC K=100 → 精细分类 → 告警决策
```

Server 夜间标定产物直接喂给 DBN:

## 7. CLI 命令参考 (cfarProcCfg 详解)

```
cfarProcCfg <subFrameIdx> <cfarMethod> <winLen> <guardLen> <noiseDivShift> <peakGroupEn> <threshold_dB> <numPeaks>
```

| 参数 | 位 | 类型 | 范围 | 说明 |
|------|---|------|------|------|
| subFrameIdx | 0 | int | 0/1 | 0=range CFAR, 1=doppler CFAR |
| cfarMethod | 1 | int | 0-2 | 0=CA, 1=CAGO, **2=CASO**(最灵敏) |
| winLen | 2 | int | 2的幂 | 噪声窗口(单侧) |
| guardLen | 3 | int | ≥0 | 保护带(单侧) |
| noiseDivShift | 4 | int | log2(winLen) | 噪声除数 |
| peakGroupEn | 5 | int | 0/1 | 峰值合并 |
| threshold_dB | 6 | float | >0 | **核心调参项，0.0会崩溃** |
| numPeaks | 7 | int | 0 | 0=不限 |

CFAR 方法: CASO(2)用 min(左,右) 做噪声估计，灵敏度最高，适合静态检测。
