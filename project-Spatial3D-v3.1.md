# 项目概要:高分辨率 3D 静态分层空间建模(V3.1)

> 本文档取代 V3.0 概要(project-Spatial3D)中与之冲突的内容。
> 核心变更:由"DSP 体素引擎 + UART 传体素图"的单通道架构,改为
> **"粗图点云 + 天线域原始切片"双通道架构**,MUSIC 等高分辨率算法
> 移至 PC 端,建图阶段不再依赖定制 DSP 固件。

---

## 1. 项目背景

本项目是养老跌倒监测系统 V3.0 的核心组成部分。目标不是实时人体检测,
而是在**房间无人或静止期间**,利用固定安装的 TI **AWRL6844** 60GHz
毫米波雷达,对室内空间进行长期累积扫描,建立稳定、可持续更新的
三维静态空间模型,为后续跌倒检测和贝叶斯推理提供空间先验(Spatial Prior)。

系统优先考虑**空间精度、长期稳定性和工程可实现性**,不要求实时建图。

### 命名与平台统一

| 项目 | 规范 |
|---|---|
| 芯片 | **AWRL6844**(L 系列,勿写作 AWR6844AOP) |
| 评估板 | AWRL6844EVM(板载 4TX/4RX 天线,RO3003 基板) |
| SDK | **MMWAVE-L-SDK**(≠ 经典 mmWave SDK,demo 名与 CLI 命令均不同) |
| 主控核 | R5F(勿写作 MSS)+ C66x DSP + HWA 1.2 |
| 开发分工 | Linux:装 SDK / 编译 / 烧录;macOS(本仓库 pc/):串口读取 / 解析 / 建模 |

## 2. 核心洞察(方案的三块基石)

### 2.1 静态场景 = 时分复用的许可证

场景静止时,每帧数据都是同一"真值集合"的独立采样。因此
**UART 单帧传不完没有关系**:按距离分块轮流传输,时间上取并集即可取全。
带宽约束被时间换掉,而"不要求实时"正是系统的既有约束。

### 2.2 静态场景 = 快拍制造机

MUSIC 等子空间算法最缺的是独立快拍数。静止房间里每一帧都是同一信号的
独立噪声实现,扫描 1 分钟即累积上千快拍,协方差矩阵可估计得非常干净。
**"夜间无人、长期累积"的约束在这里从限制变成红利。**

### 2.3 要 CFAR 之前的"脏"数据,不要 CFAR 之后的点

demo 点云是 CFAR + 聚类筛选后的产物:弱反射家具被门限滤除、
超限点按 SNR 截断(系统性丢弃弱点)、天线域相位信息已丢失。
高分辨率算法(MUSIC / Capon)需要的是 **radar cube 零多普勒切片的
天线域复数数据**(每 range bin × 16 虚拟通道)。DC 偏置、TX-RX 泄漏等
"脏"成分在 PC 端用空房间参考帧做背景扣除,比在 DSP 里硬滤更灵活。

## 3. 双通道数据架构

```
                    AWRL6844 (demo 固件,尽量零定制)
                     │
        ┌────────────┴─────────────┐
        │ 通道1:粗图               │ 通道2:细节
        │ demo 点云 TLV             │ 零多普勒天线域切片
        │ (CFAR 后检测点)           │ (range FFT 后复数快拍,
        │                           │  按 Rn~Rn+1 分块传输)
        ▼                           ▼
      UART @921600 ──────────────────
                     │
                     ▼
                PC (macOS, 本仓库 pc/)
        │                           │
        │ 体素累积 (log-odds)        │ 快拍累积 → 背景扣除
        │ → 占用图                   │ → 通道校准补偿
        │ → ROI 定位                 │ → 空间平滑 + MUSIC/Capon
        │                           │ → 高分辨角谱
        └────────────┬──────────────┘
                     ▼
          体素图回填 → RANSAC 平面拟合(墙/地/顶)
          → 连通域分析 / 形状分类(家具)
          → 语义模型 + 人工标注(门、活动区)
          → Persistent 3D Static Map(30 天滚动保存)
```

- **通道 1** 用于全屋覆盖与 ROI 定位,使用原版 demo,零固件开发。
- **通道 2** 只对 ROI 触发,配合《寻找家俱边缘.md》的
  "粗算法找 ROI → MUSIC 裁决"流程,数据流由此闭环。
- **DSP 体素引擎(V3.0 阶段 I)从关键路径退场**,降级为量产期
  无 PC 常驻场景的远期优化项。CFAR、角度估计、LUT、log-odds
  等全部智能移至 PC。

### 通道 2 带宽估算

一个 range bin 的一个零多普勒快拍 = 16 通道 × 4B(复数 int16)= 64B。
按 256 range bin 计,整切片一帧 16KB;按 32 bin 的距离分块,一帧 2KB,
921600 baud 下 10fps 仅占约 1/4 带宽。**无 SNR 截断,弱反射信息完整保留。**

## 4. 空间配置

| 参数 | 值 |
|---|---|
| 房间 | 4m × 6m × 3m |
| 体素 | 20cm × 20cm × 20cm(存储格式,不代表各维测量精度,见 §5) |
| 网格 | 20 × 30 × 15 = 9000 体素 |
| Wire format | 8B/体素:int16 logOdds + uint16 hitCount + uint16 intensity + uint8 zMin + uint8 zMax |
| 全图 | 72KB,921600 baud 下约 1s 传完(PC 侧内部格式,与 pc/spatial3d/voxel.py 一致) |

> 注:zMin/zMax 在纯 3D 网格中冗余(z 已在索引中),保留它们仅为
> 兼容 2.5D 列式视图;若确认不需要,可降至 6B/体素(54KB)。
> "180KB / 3 秒"为 V3.0 旧数据,作废。

## 5. 分维度精度预期(物理边界,禁止笼统写"5cm 精度")

| 维度 | 预期 | 依据 |
|---|---|---|
| 径向(距离) | 2~4cm | ΔR=c/2B;57-64GHz 最高 7GHz 带宽;chirp 变焦 + zoom-FFT 对孤立强点可更优 |
| 正对表面平面位置(墙/柜门) | 厘米级 | 距离精度 + RANSAC 平面拟合;角度误差仅使点沿面滑动,不影响面的位置 |
| 横向/高度边缘(近场 ≤2m) | 20~40cm | 受虚拟阵列孔径限制 |
| 横向/高度边缘(远场 ~4m,粗图) | 0.5~1m | 同上;长期累积改善 SNR,**不突破孔径衍射极限** |
| 孤立散射中心(MUSIC 后) | 角精度提升 2~5 倍(待实测) | 仅对少数离散信源模型成立;扩展面目标(整面墙)不适用 |

**工程化配套**:体素置信度按"该处横向弥散 / 体素尺寸"加权,
PC 拟合时远场点自动降权;远场家具边界靠双雷达基线(可选校准工装)、
长方体先验或人工标注收口。

> ⚠️ 待验证 P0:上表角度项引用的 15°(方位)/30°(仰角)来自
> AWR6843AOP(3T4R)。AWRL6844EVM 为 4T4R = 16 虚拟通道的 2D 阵列,
> 实际分辨率取决于虚拟阵元在两维上的分配(4×4 → 两维各约 28°;
> 8×2 → 方位 15°/仰角 58°)。**装好 L-SDK 后查天线几何文件或
> EVM 用户指南天线章节,或角反射器实测。仰角分辨率直接决定
> 床上/床下判别能力。**

## 6. 扫描调度

- **时机**:每天凌晨 2-3 点,确认无人后启动;每次约 1 分钟;
  持续累积 30 天,滚动保存最近 30 天数据。
- **距离切片轮换**:每个 session 通过 cfg 距离窗(range gating /
  boundary box 类命令,以 L-SDK 实际命令为准)只处理 Rn~Rn+1 薄壳,
  逐日轮换切片:
  - 规避 demo 检测点数上限的 SNR 截断,弱点获得输出机会;
  - 每片可独立降 CFAR 门限、独立优化 chirp(近端高斜率短 chirp,
    远端多 chirp 积累补偿 R⁴ 衰减);
  - 分块必须是**芯片端 cfg 开窗**,不能只在 PC 端被动过滤——
    否则强点每帧挤占名额,弱点永远轮不到。
- **运动中断**(保留 V3.0 的好设计):复用 demo 自带 presence/motion
  检测,检测到人/宠物即通知 PC 暂停累积(自定义 TLV 2001 或
  等价机制),避免污染静态地图。
- **传输容错**:分块数据附扫描会话 ID(epoch)与 CRC32,
  PC 端识别跨天数据与坏块,丢块只补该块。

## 7. 通道 2(MUSIC 链路)的三个前置条件

1. **相干性处理**:室内多径与目标回波相干,MUSIC 直接退化;
   需前后向空间平滑,代价是有效孔径缩水(16 阵元平滑后单维约剩一半)。
   因此 MUSIC 的收益边界:**孤立散射中心(家具棱角、桌腿、金属件)
   有效;扩展面目标(墙)无效,墙靠 RANSAC。**
2. **通道幅相校准**:MUSIC 对阵列误差极其敏感。安装流程必须包含
   角反射器标定(TI 有标准流程),标定系数存 PC 端逐帧补偿。
3. **信源数估计**:AIC/MDL + 人工上限;大量快拍可覆盖大部分不稳定性。
