## 2. cube 返回契约(2 维)

| 维度 | 值 | 含义 | 来源 |
|---|---|---|---|
| **lying** | **Y / N** | 楼下有没有倒地身体(存在)—— **单独决定 fall** | cube_ff 主 / z40 兜底 |
| **Living_state** | **Living / ?** | 活体状态(**仅标签,不是门**) | rr / body-motion |

- **"?" = 测不到,不是"死"**:典型成因是**只有腿进 cube(胸被 chair 挡)** → RR/body-motion 需要胸腔 → 测不到 → "?"。**绝不能因测不到 RR 就误报成 💔 崩溃。**
- **lying 必须能只靠腿判 Y**(胸挡住时,腿在楼下=倒地),否则遮挡跌倒会漏。

## 3. presence 判读(lying Y/N)优先级

```
cube_ff is None 且 0 条目        → 作废 · 不评估(z40 也无数据)
cube_ff >= 强阈值               → lying = Y   (cube_ff 主判据,自包含、不依赖基线)
cube_ff 弱 / =0 / 计算不出 / 多簇 → 转 z40:差值/基值 vs 空房 · XY 逐格 → z40 ≥ 阈值 ? Y : N
```

- **cube_ff = 运动能量的楼下占比**(bandpass 到运动带 → MUSIC DOA → 楼下比例);自包含,但**远/镜面/静止会失效**。
- **z40 = 差值/基值**(live vs 固定空房基线,XY 逐格);依赖基线,当兜底;**对远/遮挡/静止身体仍有效**。

### 3.1 多簇(TODO #2)
chair 隔断把身体切成 2 段(**径向**沿距离 / **轴向**沿方位),cube_ff 在两段给出**矛盾**读数 → 用 z40 仲裁。

## 4. 判决 · Fall ≥ 1

**任意一发 cube 的 `lying = Y` → 报 fall。** 理由:18s 延时已预筛走过路过;z40 正样本 34-97 vs 空房 0,一次确凿阳性即可信;要求 2 次会 +12~24s 延迟且漏报。**漏一个真崩溃 ≫ 偶尔误报。**

- Living → 红;
- "?" → 红 + 活体未知(**非崩溃**)。

## 6. 阈值(已定案)

用**昨天已录的 case/ 标注数据**标定,**不需要新采集**。

| 阈值 | **定案值** | 依据 |
|---|---|---|
| **cube_ff 强/弱** | **0.5** | 躺姿好信号(近/中有呼吸)cube_ff = 0.55/0.57/0.59/0.66/0.86/0.92;远/静止 = 0.00 —— 双峰空档,0.5 卡正中 |
| **z40 lying** | **0.4(现有,不动)** | 躺 ~28 ≫ 空 ~0,差距巨大;只判躺 vs 空 |
| 重试 X(TODO#3) | ≈60s(或 90s 保守) | 固件 300帧/300s/10% |

### cube_ff 阈值逻辑
- **cube_ff ≥ 0.5** → cube_ff 可信(近/中,有呼吸)→ 用它判 lying;
- **cube_ff < 0.5** → 太弱(远/镜面/静止)→ **转 z40**。
- cube_ff 是 `_pose_from_motion` = **运动**楼下占比,需要呼吸/动;静止或远端身体 → 0 → 交给 z40。

### z40 阈值逻辑(为什么不用抬)
- **z40 被调用时,`down` 已成立** —— 站立/走动早被**点云 Z(down 门,上游)**排掉;
- 所以 z40 **只判"躺 vs 空/鬼影",不需要区分站立** → 阈值只要压过空房噪声即可;
- 同日实测:躺 ~28、空 ~0 → **现有 0.4 足够**;拿站立(7.5)去卡 15 是多余的(站立到不了这一步)。
- z40 数值 install-dependent(靠空房基线),但"躺 vs 空"差距巨大 → 低阈值稳健。装机**录一次空房基线**即可([[no-rerecord-background]]),非每处重调参数。

### 已知物理限制(别指望)
- **cube 波束太宽(仰角秩坍缩)→ 分不了姿态(躺/坐/站),也分不了身体/家具**;静态楼下占比:躺/坐/站/家具全 ~0.67-0.76,只有空 0.31;
- **姿态 = 点云 Z(上游 down 门),不是 cube**;**排家具 = z40 差值 + 一次性空房基线**(cube 本身做不到)。
