# 空房基线漂移测试 — 2026-07-08

## 1. 背景与问题

服务器端 per-bin MUSIC 变化检测的核心是**用协方差差值**（`rel(R)=‖Rᵢ−Rⱼ‖/‖Rᵢ‖`）比较"基线"和"事件"。此前发现：

- **流内 20s**：协方差稳到 ~0.7%
- **跨会话（隔小时、中间 reset）**：失配高达 **44%**

44% 远超"人"的信号，导致跨会话根本检不出人。**必须搞清 44% 从哪来**，才能定架构（该修标定、还是修基线保鲜、还是别的）。

候选原因：
1. **soft reset**（`sensorStop → 重发 cfg → sensorStart`，生产切 profile 的动作）的相位阶跃
2. **时间**（小时级场景老化 / 硬件热漂）
3. **离散扰动**（人进出、家具被移动、冷启动重配）

---

## 2. 测试方案

### 2.1 Reset vs 时间 解耦（A–E 五段）
`reset_ab_test.py`：一条连续流里采 5 段空房，**唯一变量是"有没有 soft reset"**：

```
A  稳态（预热后，A 之前不 reset）
A→B  soft reset，等 GAP
B→C  不 reset，等 GAP        ← 与 A→B 同时长，只差那次 reset
C→D  soft reset，等 GAP
D→E  不 reset，等 GAP
```
reset 对 = (A-B, C-D)；无-reset 对 = (B-C, D-E)。指标：`rel(R)` + 均值向量方向 `dirSim`（灵敏一个量级）。

### 2.2 空房漂移时间线
把不同时间点的空房协方差两两比，看漂移随时间/扰动如何变：
- A–E：同一条流内 3.7–13.5 min（**无扰动**基准）
- A–G：19:38 vs 21:10（~92 min，中间有失败采集、reset、人进出等**扰动**）

### 2.3 坐姿检测（同流 vs 陈旧基线）
`sit_detect.py`：空房 base → soft reset → 人进来坐 → 采 sit。
- **G–H**：同一次 run 的 空房base vs 坐姿（相隔~1min，**同流**）= 干净的人信号
- **A–H**：陈旧空房(19:38) vs 坐姿(21:10) = 陈旧基线下的检测

### 2.4 捕获命名（按实际时序）
| 标签 | 物理来源 | 时间 |
|---|---|---|
| A,B,C,D,E | `reset_ab` 五段空房（同一条流） | 19:38–20:00 |
| ~~F~~ | 21:10 第一次空房 → **卡死 0 bins，废弃** | 21:10 |
| **G** | reset 后第二次空房 = sit 的**同流 base** | 21:10 |
| **H** | 坐姿 sit | 21:10 |
| baseFresh | 事后另起炉灶 reset 单采的空房（单列，不入主序列） | 21:15 |

文件：`seg_A/B/C/D/E.npz`、`seg_G.npz`、`seg_H.npz`、`seg_baseFresh.npz`（均含协方差+均值）。

## 4. 结论

### 4.1 44% 不是 soft reset，也不是（纯）时间
- **soft reset 无罪**：reset/无-reset = 1.37（3.1）
- **时间不是主因**：两次空房**相隔仅 5min** 却差 **0.48**（中间夹了卡死+xds110reset+人进出），几乎等于隔 92min 的 A-G(0.33)。**时间无法解释。**
- 同流无扰动的真实漂移**很小且平滑**：14min 才 0.058（全局）/ 0.035（椅子区）。

### 4.2 主因是「离散扰动 + 冷启动」
跨捕获失配对逐 bin 分析：**尖峰、异质（CV>1）、远近都变** → 是具体散射体被扰动（人穿过房间、碰到家具、门帘/气流），叠加 **xds110reset + 全新 cfg 冷启动**，不是均匀的时间老化或硬件相位阶跃。

### 4.3 同流能检出，跨会话检不出（决定性）
- **同流**：人信号 G-H = **0.140** ≫ 同流漂移 A-E = 0.058 → 人 **2.4×** 于漂移，**检得出**。
- **跨会话**：陈旧空房漂移 A-G = **0.328** ≫ 人信号 0.140；且 A-H(0.374) ≈ A-G(0.328) → 陈旧基线下，坐着的人和空房**几乎分不出**（全局只多 0.046）。**人被淹没。**

**判据：基线漂移必须 < 人信号(0.140) 才能检出；只有"同流/无扰动"满足。**

### 4.4 对架构的含义
1. **基线不是被"时间"废掉的，是被"离散扰动 + 冷启动"废掉的** —— 印证早先设计担忧："椅子可移动，基线也会动"。
2. **纯时间滚动基线不够**（任何时刻一次扰动即失效）。需要：
   - **事件驱动重建基线**（检测到离散变化就重采），和/或
   - **分区稳定系数**（墙=高稳、家具=低稳），和/或
   - **靠"人会呼吸、家具不会"**区分"该报警的人" vs "该更新的家具"（运动/呼吸通道）。
3. **尽量保持一条连续流**（soft reset 无害），避免 xds110reset / 冷启动重配夹在基线与事件之间。

### 4.5 尚未测准（待办）
"**纯平滑老化**"到底多大**仍未知** —— 所有长间隔捕获都被离散扰动污染。
**干净实验**：一条流不断（不 reset、不重配、持续 drain 防串口堵塞）、**房间锁死无人进出**、隔几小时每 15min 采一段空房，画纯老化曲线。若仍小 → 老化不是问题，主攻扰动检测 + 分区稳定；若变大 → 加时间驱动滚动。

## 6. 更正与补充(2026-07-09)

### 6.1 能量密度空间域是鲁棒的(修正"跨会话必失败")
- **rel(R)=44% 是逐bin协方差域**的相对失配,**不代表空间检测失败**。
- 重建+归一化到**空间能量密度**后,跨会话漂移**摊薄到每块区域仅 ~0.03**;而集中目标(人)局部 ~0.19 → **人/漂移 ≈ 5-6×,主导**。
- 前提:**必须把目标全部能量算进去**(见 6.2)。之前 report 窄带丢了躺人 94% 能量,才误判。

### 6.2 贴地目标:94% 能量入地板下 → fall zone 需含 [−2, 0.4]
- 4元俯仰孔径,**躺/贴地目标 el 估太低,~94% 能量散到地板下**(Z=−0.5~−1.9)。这不是垃圾,是"被俯仰误差搬走的人"。
- `change_report.py` 加 `--z-min/--fall-z-min`;贴地目标用 `--fall-z-min -2`,躺人从"94%丢失"变"正确检出@正确位置"。

### 6.3 X 符号标定
- 实测(人在雷达波左侧 → 能量出在 +x):**+x = 波束左 = 雷达左手边 = 面向雷达者的右手**。
- 阵元方位符号与 `awrl6844_array` 假设相反。**代码保持不变**,画图用"面向雷达"视角(雷达在上、+y向下、+x右)即正确。
- 保留:偏移小+有漂移,置信度中;贴地目标 94% 是伪影能量,方位受影响小但该用强目标重标。

### 6.4 【关键】慢时间 FFT = base-free 活体检测(micro-move)
- 每 range bin 取慢时间复序列(K帧×16天线)→ 去DC → FFT → 看非DC/呼吸带(0.15-0.5Hz)功率。
- **不用基线、不用相减、不受俯仰伪影影响**(它看频率,不看角度)。
- 实测:

| | 微动主频 | 峰/中位 |
|---|---|---|
| 空房 | 0.00Hz(DC,静态) | 4× |
| 椅子人 | 0.35Hz(21/min) | **380×** |
| 地上躺人 | 0.28Hz(17/min) | 8× |

- **区分人/空房、定距离、出呼吸率**。直立强(380×),躺人弱(8×)但呼吸峰清楚 → **躺人检测的钥匙**(静态能量看不清、俯仰伪影救不了的,呼吸能抓)。
- 工具:`slowtime_fft.png`。

## 7. 呼吸 FFT 与定位原理(2026-07-09)

### 7.1 为什么需要它:前面每一堵墙的绕过口
静态能量看不清躺人、MUSIC 角度低SNR不稳、俯仰伪影把 94% 能量搬到地板下、跨会话基线漂移 —— **呼吸只看频率,把这些全绕开**:
- 不用基线(避跨会话漂移)、不用相减(避重建噪点)、**不看角度**(避俯仰伪影)
- 人体有呼吸(0.15-0.5Hz),静物永远没有 → 天然区分活体

### 7.2 呼吸 FFT 原理
每个 range bin 取慢时间复序列(K帧 × 16天线):
```
① 去 DC(减掉帧间均值 = 去掉静态成分)
② 沿慢时间轴 FFT
③ 呼吸带(0.15-0.5Hz)功率 = 活体信号;主频 = 呼吸率
```
实测(峰/中位比):空房 **4×**(峰在DC/静态)| 椅子人 **380×**(0.35Hz=21/min)| 地上躺人 **8×**(0.28Hz=17/min)。**区分人/空房、定距离、出呼吸率。**

### 7.3 定位原理(分维度,X/Y 来源不同)
| 输出 | 公式/来源 | 精度 | 靠什么 |
|---|---|---|---|
| **Y 距离** | `Y = √(R²−H²)`(range + 地板先验 Z≈0, H=2m) | ±0.18m | range-FFT(2.34cm/bin)|
| **X 横向** | `X = Y·tan(az)`(呼吸带信号做波束/角度-FFT) | ±0.10m | 呼吸窄带 SNR 增益 |
| **活体/率** | 慢时间 FFT 呼吸带 | — | 慢时间-FFT |

- **跌倒贴地(Z≈0)→ 距离直接给 Y,不碰不可靠的俯仰。**
- 实测 cal_I:X=+0.56±0.10m, Y=3.44±0.18m(6 bin 高度一致)。

### 7.4 【关键】准确性来自 FFT,不是 MUSIC
在呼吸带信号上,**MUSIC 与 Bartlett(=FFT波束)方位完全相同**(都 +10°, X=+0.57±0.10m)。
- **单目标定位靠的是"呼吸窄带滤波的相干增益(SNR)",不是 MUSIC 超分辨。**
- MUSIC/超分辨只在**分辨多个近距目标(<1m)**时才需要;单个跌倒用不着。
- → **单目标可纯 FFT 落地**(实时端不用跑 MUSIC)。

### 7.5 三级 FFT 流水线(单目标,无 MUSIC)
```
①range-FFT(快时间) → 距离 → Y(+地板先验)
②慢时间-FFT         → 呼吸带 → 哪个bin有活人 + 呼吸率 + 活体
③角度-FFT/波束(天线)→ 方位 → X
```
**Cube 是这三个 FFT 的原料(慢时间×天线);角度那步不需要 MUSIC。**

### 7.6 为什么标准雷达"跑偏",这条稳
| | 标准雷达(跑偏) | 本方案(稳) |
|---|---|---|
| 时间 | **单帧快照**(角度方差∝1/SNR) | 积分~40s(呼吸)/累积协方差 |
| 观测量 | range-**Doppler** CFAR(**漏静态**) | zero-Doppler + 呼吸窄带 |
| 距离 | FFT(其实一直准) | FFT + 地板先验 |
| 角度 | 单帧角度-FFT(抖) | 窄带增益后波束(稳) |

跑偏 = "在最弱信号、最少信息(单帧)、最难目标(静止)下判断"。**稳定的钥匙 = 积分 + 呼吸窄带 + 距离/地板先验,把每帧的抖平掉。**

### 7.7 架构地基:5fps → 慢传 → 原始 cube
```
5fps(延迟不敏感,免费) → 每帧UART预算翻倍 → 184bins全房间一帧
  → 吐 zero-Doppler 天线复数(全房间、逐帧) → 原始 cube[bins×K×16]
  → 后面呼吸FFT/协方差/MUSIC/能量密度 全是它的服务器端视图
```
标准雷达只给**有损、每帧、靠多普勒的点云** → 拿到手只能跟着跑偏,回不了头。
**原始 cube = 全信息原料** → 才能跨帧积分、抠呼吸、跨会话鲁棒,且算法全服务器端更新、不重刷固件。
**cube 是必要地基;把它变成稳定定位的是"积分+呼吸窄带+地板先验"那套后处理 —— 缺一不可。**
