# 空房基线漂移测试 — 2026-07-08

## 1. 背景与问题

服务器端 per-bin MUSIC 变化检测的核心是**用协方差差值**（`rel(R)=‖Rᵢ−Rⱼ‖/‖Rᵢ‖`）比较"基线"和"事件"。此前发现：

- **流内 20s**：协方差稳到 ~0.7%
- **跨会话（隔小时、中间 reset）**：失配高达 **44%**

44% 远超"人"的信号，导致跨会话根本检不出人。**必须搞清 44% 从哪来**，才能定架构（该修标定、还是修基线保鲜、还是别的）。

候选原因：
1. **soft reset**（`sensorStop → 重发 cfg → sensorStart`，生产切 profile 的动作）的相位阶跃
2. **时间**（小时级场景老化 / 硬件热漂）
3. **离散扰动**（人进出、家具被移动、冷启动重配）

---

## 2. 测试方案

### 2.1 Reset vs 时间 解耦（A–E 五段）
`reset_ab_test.py`：一条连续流里采 5 段空房，**唯一变量是"有没有 soft reset"**：

```
A  稳态（预热后，A 之前不 reset）
A→B  soft reset，等 GAP
B→C  不 reset，等 GAP        ← 与 A→B 同时长，只差那次 reset
C→D  soft reset，等 GAP
D→E  不 reset，等 GAP
```
reset 对 = (A-B, C-D)；无-reset 对 = (B-C, D-E)。指标：`rel(R)` + 均值向量方向 `dirSim`（灵敏一个量级）。

### 2.2 空房漂移时间线
把不同时间点的空房协方差两两比，看漂移随时间/扰动如何变：
- A–E：同一条流内 3.7–13.5 min（**无扰动**基准）
- A–G：19:38 vs 21:10（~92 min，中间有失败采集、reset、人进出等**扰动**）

### 2.3 坐姿检测（同流 vs 陈旧基线）
`sit_detect.py`：空房 base → soft reset → 人进来坐 → 采 sit。
- **G–H**：同一次 run 的 空房base vs 坐姿（相隔~1min，**同流**）= 干净的人信号
- **A–H**：陈旧空房(19:38) vs 坐姿(21:10) = 陈旧基线下的检测

### 2.4 捕获命名（按实际时序）
| 标签 | 物理来源 | 时间 |
|---|---|---|
| A,B,C,D,E | `reset_ab` 五段空房（同一条流） | 19:38–20:00 |
| ~~F~~ | 21:10 第一次空房 → **卡死 0 bins，废弃** | 21:10 |
| **G** | reset 后第二次空房 = sit 的**同流 base** | 21:10 |
| **H** | 坐姿 sit | 21:10 |
| baseFresh | 事后另起炉灶 reset 单采的空房（单列，不入主序列） | 21:15 |

文件：`seg_A/B/C/D/E.npz`、`seg_G.npz`、`seg_H.npz`、`seg_baseFresh.npz`（均含协方差+均值）。

## 3. 结果

### 3.1 Reset 无罪（A–E）
| 配对 | 类型 | rel(R) | dirSim |
|---|---|---|---|
| A-B | RESET | 0.030 | 0.9998 |
| C-D | RESET | 0.015 | 1.0000 |
| B-C | 无reset | 0.018 | 0.9999 |
| D-E | 无reset | 0.014 | 1.0000 |

→ **soft reset 无罪**（reset 对 ≈ 无-reset 对，都 ~2%）。

## 4. 结论

### 4.1 44% 不是 soft reset，也不是（纯）时间
- **soft reset 无罪**：reset/无-reset = 1.37（3.1）
- **时间不是主因**：两次空房**相隔仅 5min** 却差 **0.48**（中间夹了卡死+xds110reset+人进出），几乎等于隔 92min 的 A-G(0.33)。**时间无法解释。**
- 同流无扰动的真实漂移**很小且平滑**：14min 才 0.058（全局）/ 0.035（椅子区）。

### 4.2 主因是「离散扰动 + 冷启动」
跨捕获失配对逐 bin 分析：**尖峰、异质（CV>1）、远近都变** → 是具体散射体被扰动（人穿过房间、碰到家具、门帘/气流），叠加 **xds110reset + 全新 cfg 冷启动**，不是均匀的时间老化或硬件相位阶跃。

### 4.3 同流能检出，跨会话检不出（决定性）
- **同流**：人信号 G-H = **0.140** ≫ 同流漂移 A-E = 0.058 → 人 **2.4×** 于漂移，**检得出**。
- **跨会话**：陈旧空房漂移 A-G = **0.328** ≫ 人信号 0.140；且 A-H(0.374) ≈ A-G(0.328) → 陈旧基线下，坐着的人和空房**几乎分不出**（全局只多 0.046）。**人被淹没。**

**判据：基线漂移必须 < 人信号(0.140) 才能检出；只有"同流/无扰动"满足。**

### 4.4 对架构的含义
1. **基线不是被"时间"废掉的，是被"离散扰动 + 冷启动"废掉的** —— 印证早先设计担忧："椅子可移动，基线也会动"。
2. **纯时间滚动基线不够**（任何时刻一次扰动即失效）。需要：
   - **事件驱动重建基线**（检测到离散变化就重采），和/或
   - **分区稳定系数**（墙=高稳、家具=低稳），和/或
   - **靠"人会呼吸、家具不会"**区分"该报警的人" vs "该更新的家具"（运动/呼吸通道）。
3. **尽量保持一条连续流**（soft reset 无害），避免 xds110reset / 冷启动重配夹在基线与事件之间。

### 4.5 尚未测准（待办）
"**纯平滑老化**"到底多大**仍未知** —— 所有长间隔捕获都被离散扰动污染。
**干净实验**：一条流不断（不 reset、不重配、持续 drain 防串口堵塞）、**房间锁死无人进出**、隔几小时每 15min 采一段空房，画纯老化曲线。若仍小 → 老化不是问题，主攻扰动检测 + 分区稳定；若变大 → 加时间驱动滚动。
