# 设计结论 — 静态 3D 空间模型与跌倒检测(2026-07-05 会话)

本文件汇总本次会话达成的核心观点与结论,供后期回看。配套的可执行环境细节见
[GET-DATA-FROM-TI.md](GET-DATA-FROM-TI.md)。

## 0. 一句话总纲

> **本系统是一个"耐心的 3D 静态变化监测器":离线慢慢建一张尽可能高清的静态背景图 A,
> 一切事件(含跌倒)都是"相对 A 的持续异常"——用位置(A 本身)判别性质,用时间累积置信度。**
> 它对雷达的粗分辨率和不可靠的实时 track 天然免疫,因为它一个都不依赖。

## 1. 硬件与几何现状(已查证)

- 芯片:**TI AWRL6844**(AoP,60GHz,L 系列,用 MMWAVE-L-SDK 06.00.04.01)
- 虚拟阵:**4×4 = 16 阵元**,λ/2 间距,**方位/俯仰对称**
  - `antGeometryTX 2 2 2 0 0 0 0 2` / `antGeometryRx 1 0 0 0 0 1 1 1` / `antGeometryDist 2.54 2.54`
  - 每轴仅 4 阵元 → FFT 波束宽 ≈ **25°** → 2.5m 处约 **1.1m 宽波束**
  - 实测定位误差 **50~60cm**(粗大角度模糊 + 杂波/多径导致)
- 距离分辨率:**厘米级**(靠 60GHz 宽带 chirp)—— 距离维不是瓶颈,**角度维才是**
- 安装几何(固定):**高 2.3m,俯仰 35°偏离垂直**(高处俯视)
  - 关键推论:此几何下 **俯仰角 ≈ 目标高度**,所以锐化俯仰 = 直接锐化高度
  - 桌面(0.75m)与人躯干(1.2m)在 2.5m 处俯仰仅差 ~10° < 25° 波束 → **分不开 → track 飘在桌面上方**

## 3. 核心架构决策

- **静态 3D 模型本身就是要的东西**,其余(跌倒、人员定位…)都是**它之上的查询**,而非"追踪一个点"。
- 数据结构:`pc/` 现有 `VoxelGrid`(20×30×15,每体素带 intensity)即天然容器。
  - 某 (x,y) 立柱的 **Z 能量密度** = 该列 15 个 Z 体素的 intensity
  - **Z≤40cm 地面带** = 最底 2 个体素

## 4. 跌倒检测方法(用户核心思路)

- 跌倒 = 能量从"站立高柱(~150cm)"塌成"贴地低团(≤40cm)"——**粗大重分布,对粗分辨率免疫**
- 二段式(但见 §5 后进一步简化):
  1. 粗触发:track stuck/lost + 能量砸向 ≤40cm
  2. **确认:人跌倒后变静态 → 对 Fall bin 做 gazing 累积 → 得清晰局部图 B → B−A 差 = 贴地新质量 → 高置信度**
- **图 A 就是判别器**:躺**床上**(A 里床在那)= 正常;躺**开阔地面**(A 里是自由地面)= 异常
- 必须做**能量域背景相减**:地面带本有基线能量(地板/床底),看的是 **ΔE_floor 相对 A 的增量**

## 5. 关键约束:延迟不敏感 → 架构再塌缩

- **30s vs 300s 对急救无差别**(研究:即使摄像头,到场也仅 15min→9min;瓶颈在物理到场)
- ⇒ **为置信度优化,不为速度**。可**丢掉脆弱的实时触发和 tracker**。
- 核心退化为:**基线 A(时间常数 小时~天) + 分钟级当前图 → 差分 → 持续的、人体大小的、贴地新质量,
  停留数分钟、不在床/沙发位、期间无"起身" = 跌倒**
- **"持续性/时间"= 置信度轴 = 误报天然克星**(真跌倒起不来;假触发扛不过几分钟)
- 时间常数分层:A 吸收变化远慢于报警触发 → 刚跌的人绝不会被基线吸收掉

## 6. 核心投资:高清 Map A(技术脊梁)

- **A 的清晰度封顶整个系统精度**——糊的 A 去比对,残差全是 A 自己的噪声,人形信号淹没。
  **所以超分辨对 A 不是可选,是核心。**(纠正会话早期"超分辨可选"的说法)
- **自洽点**:正因"延迟不敏感 + A 离线慢建",所有重武器才用得起:
  | 工具 | 给 A 的贡献 |
  |---|---|
  | 长期多夜累积 | 压噪声、升占用置信度(即使角分辨率不变,图也干净得多) |
  | **超分辨(MUSIC / IAA / 稀疏重建)** | 60cm → ~20-30cm;需**改固件吐复数**,"洋葱"逐层慢流正好配离线 A |
  | 走圈轨迹(干净多普勒) | 用可靠动目标描家具**锐利 footprint**,补静态检测短板 |
- **算法选型**(静态+稀疏+相干场景):**IAA / 稀疏重建 可能优于 MUSIC**
  (MUSIC 需空间平滑解相干、损孔径;IAA/CS 单快照、天然抗相干、不损孔径;固件端要吐的复数数据三者相同)
- **多径鬼影**:超分辨救不了,需多普勒一致性 + 几何约束的鬼影剔除(与超分辨正交)

## 7. 分辨率天花板(诚实预期)

- 4×4 阵是硬极限:超分辨 + 累积能把 **60cm(糊)→ ~20-30cm(干净)**,**到不了厘米级**(厘米级只在距离维)。
- 够分清床沿/地面、人/物;不够时才轮到**多传感器融合**(多角度补孔径 + 多径分集)。

## 8. 推进计划(两阶段)

1. **先搭累积+差分骨架**(用现有点云):立起 Map A 累积器、分钟级当前图、B−A 差分、
   **一个"A 清晰度"量化指标**——先看见 A 现在多糊,才知超分辨要补多少,也给高清结果一个容器。
2. **再攻高清 A**:改固件吐复数 → PC 端 IAA/MUSIC/稀疏 → 融合走圈轨迹 → 多夜累积。
